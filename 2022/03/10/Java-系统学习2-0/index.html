8.1 StringBuffer StringBuilder的区别

  1. 相同之处：字符串缓冲区（字符串序列可以更改的字符串），并且API兼容（方法相同）

  2. 版本不同：前者1.0，后者1.5

8.2 String StringBuffer StringBuilder的区别

String：字符串常量

StringBuffer、StringBuilder：字符缓冲区

相同之处：都来描述字符串

不同之处：String是字符串常量，对象一旦创建，字符序列不能更改；StringBuffer、StringBuilder是字符串缓冲区，字符序列可以更改

使用场景：如果频繁使用字符串拼接，建议使用字符串缓冲区；如果没有链接和截取等操作，就用字符串常量

处理异常方式：

  1. 抛出机制

    思路：把方法出现的异常封装为对象 抛给方法的调用者
    格式：在方法声明上添加 throws 异常类型 来把可能出现的异常抛给方法调用者
    类似于：商品上贴的标签(残次品，可能过期)
    作用：

    • 当前方法可能出现那些类型的异常

    • 如果出现这些异常 当前方法没有处理 而是把异常分装为对象 交给方法的调用者处理

    注意事项：

    1. throws 后面可以加多个异常

    2. 方法通过抛出机制处理异常 如果方法出现异常 方法结束 当前方法下面的代码不再执行

    3. 如果都选择抛出机制处理异常 最终抛给虚拟机：虚拟机的统一处理方式：先通过错误输出流打印异常信息，然后停止虚拟机

    4. 子类重写父类的方法：子类方法不能抛出比父类方法多的 大的异常

  2. 捕获机制

    思路：方法通过try-catch代码块 把出现的异常情况封装为对象 并处理掉
    格式：

注意事项：

​ 1 当try代码块出现异常 try代码块中其他代码不再执行 而是执行对应的catch代码块
​ 2 捕获父类异常的catch代码块 必须放在捕获子类异常catch代码块的后面

自定义异常

  1. 创建一个类，继承Throwable 或者其子类
  2. 创建有参数的构造方法，通过参数来指定异常原因
  3. 在构造方法中调用父类有参数的构造方法，参数用于传递异常原因
  4. 在指定情况下，通过throw 自定义异常对象产生异常情况
  5. 选择处理异常方式，建议捕获（自定义异常）。

异常分类：
运行时异常（未检查异常）：继承了RunTimeException异常
编译时异常（已检查异常）：没有继承RunTimeException

  2. 编译器是否检查：

    如果方法体可能出现编译时异常，编译器会检查，要求处理

    如果方法体可能出现运行时异常，编译器不检查如果不处理，相当于抛出机制

  3. 出现的频率：
    编译时异常，只有在特定的情况下，才可能出现，出现的频率低
    运行时异常：出现频率高

10.3.2 重写HashCode和equals的原理

Hashcode返回的是内存地址（唯一的），equals对比的是对象的相关字段。

  • 当内存地址一样的时候，这个对象和其他的所有相同hashcode的对象相比，用equals比较，只要有一个equals是相同，返回true，两个true说明一样对象。

  • 当内存地址一样的时候，若是equals方法里的字段不相同，返回false，说明两个对象不一样。

  • 当内存地址不一样时，直接说明不一样的对象。

注意：equals要判断类型

13. 分类

按照方向分类：

  • 输入流（源文件）：相对于内存而言，把存储设备上的信息读到内存中 （源文件读到内存）

  • 输出流（目的文件）：相对于内存而言。把内存上的信息写道外部存储设备（内存中的写到硬盘中）

按照操作基本单位分类：

  • 字节流: 流对象操作数据基本单位是字节，适用于所有类型的信息

命名：XXXStream

学习：FileInputStream，FileOutputStream

  • 字符流：流对象操作数据基本单位是字符，只适用于文本类型的信息

命名：XXXWriter/XXXReader

学习：FileReader+FileWriter

5. Flush

输入流没有flush，因为它没有缓冲区,只有输出流有flush

除了FileOutputStream，其他的输出流都有缓冲区（把信息写入缓冲区中），都需要flush（把缓冲区的信息刷新到目的文件中）

注意：虽然有些输出流会自动flush，但是如果不flush，一次写入，占内存，如果每次读每次flush，所用的内存就小的多，效率高

13.4 序列化流

序列化流： ObjectInputStream+ObjectOutputStream

作用：实现内存中兑现的持久化保存和构建，实现对象的持久存储（比如电脑关机）

使用场景：

电脑关机，需要重写写一个一模一样的对象。关机前，对象实现序列化，将对象信息从内存中写入硬盘上，进行持久化存储，因此ObjectOutputStream是序列化过程；电脑第二页天重新打开，我开始将对象从硬盘中读取到内存中，这个过程叫反序列化（ObjectInputStream）

1. ObjectOutputStream：序列化（持久性存储）

1.1ObjectOutputStream(OutputStream out)
1.2 void writeObject（Object obj）

一次把一个对象的字节信息写到目的文件中

2. ObjectInputStream：反序列化

Object readObject();一次读一个对象，当抛出异常时，读取操作停止，因此通过捕捉，实现停止

3.注意

注意1：类必须实现序列化接口：Serializable:::保证序列化和反序列化时使用的类是同一个
